```markdown
# Docker — инструкция

Этот документ объясняет, как запускать проект с помощью Docker.

Сборка и запуск (docker-compose)
--------------------------------
1. Построить и запустить:
   docker-compose up --build

2. Запустить в фоне:
   docker-compose up --build -d

3. Остановить:
   docker-compose down

Как это работает
----------------
- entrypoint.sh проверяет, есть ли файлы в /app/models. Если нет — запускает src/train.py, чтобы обучить модели и сохранить их в /app/models и метрики в /app/outputs.
- Затем стартует uvicorn (FastAPI) на порту 8000.

Доступ
------
- Swagger UI: http://localhost:8000/docs
- Redoc: http://localhost:8000/redoc

Сохранение артефактов
--------------------
Мы монтируем локальные папки ./models и ./outputs в контейнер, чтобы модели не терялись при пересоздании образов.

Примечания
---------
- Если хотите принудительно переобучить модели — удалите локальную папку models/* и перезапустите контейнер.
- Для отладки можно исполнить bash внутри контейнера:
  docker-compose exec app bash
```
